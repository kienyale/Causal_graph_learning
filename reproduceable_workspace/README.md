# Reproducible LightGCN Workflow

This workspace contains the code, raw data, cached CSV summaries, figures, and tables for `../final_report.tex`. See the repository-level `README.md` for the full reproduction guide.

## Data

The Coat benchmark is stored in `data_raw/coat/`. Parsed arrays are regenerated into `artifacts/data_cache/` by `scripts/run_coat_lightgcn.py --artifacts-only` or by a full training run.

Yahoo! R3 Kaggle mirror files are present under `data_raw/yahoo_r3/kaggle/`:

```text
data_raw/yahoo_r3/kaggle/user.txt
data_raw/yahoo_r3/kaggle/random.txt
```

The official Webscope filenames are still supported if obtained:

```text
data_raw/yahoo_r3/ydata-ymusic-rating-study-v1_0-train.txt
data_raw/yahoo_r3/ydata-ymusic-rating-study-v1_0-test.txt
```

The copied RL4Rec repositories and matrices are not part of this cleaned submission. They were removed because the report uses the raw Coat files and the Yahoo train/test pair above.

## Dependencies

Use Python 3. The artifact/figure scripts require `numpy`, `pandas`, `matplotlib`, `seaborn`, and `scipy`. Full retraining additionally requires `torch` and a CUDA-enabled PyTorch install; the Coat training script exits if CUDA is unavailable. The report PDF is compiled with `pdflatex`.

## Run Experiments

Run the full CUDA experiment:

```bash
python3 scripts/run_coat_lightgcn.py --seed-count 20
```

The script intentionally raises an error if CUDA is unavailable. The final report uses 120 runs: 6 model variants across seeds 0--19. The cached metrics record about 18.4 minutes of training-loop time on the RTX 3060; a full end-to-end rerun can take longer because evaluation, checkpoint writing, and figure generation are outside that timer.

## Regenerate Tables And Figures

Regenerate plot-ready tables and publication figures from cached results:

```bash
python3 scripts/run_coat_lightgcn.py --artifacts-only
python3 scripts/make_mechanism_artifacts.py
python3 scripts/make_report_figures.py
```

Use the quick smoke test only for debugging:

```bash
python3 scripts/run_coat_lightgcn.py --quick
```

Run the Yahoo! R3 20% smoke test from the downloaded Kaggle mirror:

```bash
python3 scripts/run_yahoo_lightgcn_smoke.py --source kaggle --sample-frac 0.20 --seed 0 --epochs 20 --device cuda
```

Run the full Yahoo! R3 one-seed replication:

```bash
python3 scripts/run_yahoo_lightgcn_smoke.py --source kaggle --output-name yahoo_full --sample-frac 1.0 --seed 0 --epochs 20 --device cuda
```

To audit whether the required Yahoo files are present:

```bash
python3 scripts/run_yahoo_lightgcn_smoke.py --audit-only --source kaggle
```

## Outputs

Cached run outputs are saved under `artifacts/runs/coat/<model>/seed_<seed>/`. The cleaned repository keeps `config.json`, `metrics.json`, `history.csv`, `per_user_metrics.csv`, and `topk_recommendations.csv`. Model checkpoints and learned embeddings are regenerated by full training runs and are intentionally not tracked.

Report source tables are in `artifacts/summaries/` and `tables/`. Report figures are in `figures/` as both PDF and PNG. The mechanism analysis files are `mechanism_diagnostics.csv`, `swap_summary.csv`, `bucket_recall.csv`, and `fig5_mechanism.*`.

Yahoo full-data outputs are in `artifacts/yahoo_full/`, `figures/yahoo_full/`, and `tables/yahoo_full/`.

The executed audit notebook is:

```text
notebooks/01_reproduce_coat_lightgcn.ipynb
```
